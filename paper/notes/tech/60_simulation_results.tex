\section{Simulation notes: grid evaluation, cutoffs, and action rates}\label{sec:sim_notes}

\subsection{Purpose and role of the simulation}\label{subsec:sim_purpose}

The analytical results characterize best responses through the incentive difference
\[
    \Delta(v;S,\delta) \equiv u_1(v;S,\delta) - u_0(v;S),
    \qquad
    a^\star(v;S,\delta)=\mathbf{1}\{\Delta(v;S,\delta)\ge 0\}.
\]
The \texttt{computational/} code supports the theory by numerically evaluating $\Delta$ and $a^\star$ on a dense
grid in $(v,\log S,\delta)$ in order to (i) visualize comparative statics, (ii) compress outcomes into cutoffs and
action rates, and (iii) diagnose when $a^\star(v)$ behaves like a single-crossing cutoff.

\subsection{How the model is simulated (core model \& helpers)}\label{subsec:sim_how}

All simulation code is implemented in R and organized into:
\begin{itemize}
    \item \textbf{Core model functions} in \texttt{computational/R/functions/model.R}. These define $u_0(\cdot)$,
    $u_1(\cdot)$, and $\Delta(\cdot)$ (exposed as \texttt{Delta\_incentive()}).
    \item \textbf{Grid construction helpers} in \texttt{computational/R/functions/grids.R}. These create evaluation
    grids over $v$ and over $(\log S,\delta)$.
    \item \textbf{Manifest / reproducibility helpers} in \texttt{computational/R/functions/manifest.R}. Runners write
    JSON manifests recording settings and output hashes/sizes.
\end{itemize}

I interpret candidate valence as distributed
\[
    v \sim \mathcal{N}(0,\sigma^2),
\]
and use this distribution only for aggregation and summary statistics. On a finite grid, extreme tails of $v$ are
both empirically less relevant and a source of numerical instability when computing probability ratios and
differences. Using a normal reference distribution focuses computation on the region of $v$ where most mass lies and
yields stable numerical integrals for implied action rates. I keep $\sigma$ explicit as a scale parameter and choose
$v$-ranges in multiples of $\sigma$ (e.g.\ $v\in[-4\sigma,4\sigma]$).

I grid over $\log S$ rather than $S$ directly. This enforces $S>0$ and samples $S$ multiplicatively (equal steps in
$\log S$ correspond to equal percentage changes in $S$), which is more numerically stable over wide ranges.

\paragraph{Step 1: evaluate incentives and best responses on a grid.}
The runner \texttt{computational/R/runners/01\_single\_agent\_grid.R} constructs (i) a dense $v$ grid on a bounded
interval and (ii) a rectangular grid over $(\log S,\delta)$. For each $(\log S,\delta)$ cell it evaluates
$\Delta(v;S,\delta)$ over the $v$ grid and stores the implied best response $a^\star(v;S,\delta)$ using a fixed
tie-break convention. Output: \texttt{computational/output/datasets/single\_agent\_grid.csv}.

\paragraph{Step 2: compress to cutoffs and action rates.}
The runner \texttt{computational/R/runners/02\_cutoffs\_and\_action\_rates.R} summarizes each $(\log S,\delta)$ cell
with:
\begin{itemize}
    \item a cutoff $v^\star(\log S,\delta)$ defined as the first $v$ on the grid where $\Delta(v;S,\delta)\ge 0$
    (with $v^\star=+\infty$ if no such $v$ exists), and
    \item an implied action rate $\Pr(a^\star=1\mid \log S,\delta)$ computed by numerically integrating
    $a^\star(v;S,\delta)$ under $v\sim\mathcal{N}(0,\sigma^2)$.
\end{itemize}
The script also counts the number of switches in $a^\star(v)$ along the ordered $v$ grid to diagnose whether the best
response is consistent with a single cutoff. Output: \texttt{computational/output/derived/cutoffs\_and\_action\_rates.csv}.

\paragraph{Step 3: targeted diagnostics for non-monotone cells.}
The analysis script \texttt{computational/R/analysis/03\_diagnose\_nonmonotone\_cells.R} focuses on cells where
$a^\star(v)$ switches two or more times. It checks robustness to small numerical tolerances and can recompute
$\Delta(v)$ on a denser $v$ grid for selected cells. Outputs are written under \texttt{computational/output/diagnostics/}.

\subsection{How to read the numerical output}\label{subsec:sim_interpret}

The simulation output is designed to answer two questions: (i) is the best response approximately a single cutoff in
$v$ for each $(\log S,\delta)$ cell, and (ii) how often is $a=1$ chosen in a heterogeneous population of candidates?

\paragraph{Regimes.}
Each $(\log S,\delta)$ cell is classified by the pattern of $a^\star(v)$ across the $v$ grid:
\begin{itemize}
    \item \texttt{always0}: $a^\star(v)=0$ for all $v$ on the grid (the action is dominated on the evaluated range);
    \item \texttt{always1}: $a^\star(v)=1$ for all $v$ on the grid (the action is optimal on the evaluated range);
    \item \texttt{cutoff\_ok}: exactly one switch in $a^\star(v)$ (a single cutoff $v^\star$ summarizes the cell);
    \item \texttt{nonmonotone}: two or more switches (the single-cutoff summary is not informative on its own).
\end{itemize}

\paragraph{Action rates.}
The implied action rate $\Pr(a^\star=1\mid \log S,\delta)$ integrates the best response over
$v\sim\mathcal{N}(0,\sigma^2)$. It remains well-defined regardless of whether $a^\star(v)$ is a clean cutoff, and it
is the main quantitative object used to summarize behavior across $(\log S,\delta)$.

\subsection{Simulation results (action-rate and cutoff summaries)}\label{subsec:sim_results}

Below is a summary of numerical results on the baseline grid. All figures referenced below are generated by
\texttt{computational/R/analysis/results\_cutoffs\_action\_rates.R} and saved under
\texttt{computational/output/results/figures/}.

\paragraph{Cutoff logic holds for almost all parameter cells.}
Across the $(\log S,\delta)$ grid (1681 cells), the best response is overwhelmingly either constant in $v$ or a
single-switch cutoff. Regime shares are:
\begin{itemize}
    \item \texttt{always0}: 572 cells (34.0\%);
    \item \texttt{always1}: 80 cells (4.76\%);
    \item \texttt{cutoff\_ok}: 1010 cells (60.1\%);
    \item \texttt{nonmonotone}: 19 cells (1.13\%).
\end{itemize}
Figure~\ref{fig:sim_regime_map} plots these regimes over $(\log S,\delta)$ and highlights that multi-switch behavior
is confined to a small subset of the grid.

\begin{figure}[t]
    \centering
    \includegraphics[width=0.85\textwidth]{computational/output/results/figures/regime_map.png}
    \caption{Regime map over $(\log S,\delta)$: always-$0$, always-$1$, single-switch cutoff, and nonmonotone cells.}
    \label{fig:sim_regime_map}
\end{figure}

\paragraph{Action rates are polarized across the grid.}
The implied action rate $\Pr(a^\star=1\mid \log S,\delta)$ is highly dispersed:
the median action rate is $9.0\times 10^{-4}$; the 75th percentile is 0.101; the 90th percentile is 0.875; and the
95th percentile is 0.999. Figure~\ref{fig:sim_action_rate_heatmap} shows the full action-rate surface over
$(\log S,\delta)$.

\begin{figure}[t]
    \centering
    \includegraphics[width=0.85\textwidth]{computational/output/results/figures/heatmap_action_rate.png}
    \caption{Implied action rate $\Pr(a^\star=1\mid \log S,\delta)$ over the simulation grid.}
    \label{fig:sim_action_rate_heatmap}
\end{figure}

\paragraph{Cutoff surface and numerical accuracy of cutoff-based aggregation.}
When a single-switch cutoff applies and the script reports a cutoff-implied action rate
(\texttt{action\_rate\_cutoff} is non-missing; 1662 cells), the cutoff-implied and numerically integrated action
rates are very close: the median absolute difference is $2.69\times 10^{-5}$, the 90th percentile is
$8.93\times 10^{-4}$, and the maximum is $1.30\times 10^{-3}$. The implied cutoff surface is shown in
Figure~\ref{fig:sim_vstar_heatmap} (with $v^\star=+\infty$ masked and the color scale capped for readability).

\begin{figure}[t]
    \centering
    \includegraphics[width=0.85\textwidth]{computational/output/results/figures/heatmap_v_star.png}
    \caption{Cutoff surface $v^\star(\log S,\delta)$ (capped for readability; cells with $v^\star=+\infty$ are masked).}
    \label{fig:sim_vstar_heatmap}
\end{figure}


For each $\log S$, the results script records the smallest $\delta$ at which the model is \texttt{always0} and the
largest $\delta$ at which it is \texttt{always1} (see \texttt{computational/output/results/tables/frontiers\_by\_logS.csv}).
The \texttt{always0} frontier $\delta_{\text{always0}}(\log S)$ rises with $\log S$ over the explored range (e.g.,
0.52 at $\log S=-4$, 1.78 at $\log S=0$, and 2.50 at $\log S=1.8$), consistent with the interpretation that when the
opponent is stronger (higher $S$), the nomination gain from action $a=1$ can justify a larger general-election
penalty $\delta$.

\paragraph{Caveat: regime shares depend on the chosen grid.}
The reported fractions of \texttt{always0} and \texttt{always1} cells are properties of the finite
$(\log S,\delta)$ grid (in particular $\delta_{\max}$). Extending the grid to higher $\delta$ mechanically increases
the number of \texttt{always0} cells. For this reason, the frontier objects and the action-rate surface
(Figure~\ref{fig:sim_action_rate_heatmap}) are more informative than raw regime shares.

\subsection{Outputs and file locations}\label{subsec:sim_outputs}

The pipeline produces raw outputs, derived summaries, and paper-ready results tables/figures.

\paragraph{Raw grid output (large, local).}
\begin{itemize}
    \item \texttt{computational/output/datasets/single\_agent\_grid.csv}: raw grid evaluations of
    $(u_0,u_1,\Delta,a^\star)$ over $(v,\log S,\delta)$. This file can be very large; the main workflow uses it only
    for targeted inspection (e.g.\ plotting $\Delta(v)$ and $a^\star(v)$ for selected cells) and for debugging.
\end{itemize}

\paragraph{Derived per-cell summaries (small).}
\begin{itemize}
    \item \texttt{computational/output/derived/cutoffs\_and\_action\_rates.csv}: per-cell summary indexed by
    $(\log S,\delta)$ including $v^\star$, implied action rates, and switch-count diagnostics.
\end{itemize}

\paragraph{Results tables and figures for these notes.}
Running \texttt{computational/R/analysis/results\_cutoffs\_action\_rates.R} produces:
\begin{itemize}
    \item Tables under \texttt{computational/output/results/tables/}, including:
    \texttt{grid\_info.csv},
    \texttt{regime\_counts.csv},
    \texttt{switch\_count\_distribution.csv},
    \texttt{action\_rate\_quantiles.csv},
    \texttt{v\_star\_summary.csv},
    \texttt{cutoff\_approx\_error\_summary.csv},
    \texttt{cutoff\_approx\_error\_by\_regime.csv}, and
    \texttt{frontiers\_by\_logS.csv}.
    \item Figures under \texttt{computational/output/results/figures/}:
    \texttt{heatmap\_action\_rate.png},
    \texttt{heatmap\_v\_star.png}, and
    \texttt{regime\_map.png}.
\end{itemize}

\paragraph{Optional diagnostics.}
\begin{itemize}
    \item \texttt{computational/output/diagnostics/nonmonotone\_cells\_diagnostics.csv} and
    \texttt{computational/output/diagnostics/figures/}: targeted diagnostics for cells in which $a^\star(v)$ switches
    multiple times.
\end{itemize}

\paragraph{Manifests.}
\begin{itemize}
    \item \texttt{computational/output/manifests/*.json}: manifests recording run parameters and hashes/sizes of
    generated outputs.
\end{itemize}
